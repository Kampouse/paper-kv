# paper-kv — Paper Trading Bot (Python)

Paper trading with real prices from NEAR Intents, results stored on-chain via FastData KV. Python stdlib only — no pip, no npm.

## Location

`~/.openclaw/workspace/paper-kv/` — repo: `Kampouse/paper-kv`

## Quick Commands

```bash
cd ~/.openclaw/workspace/paper-kv

# Setup: registers OutLayer wallet, writes .env, runs bot
bash setup.sh run

# Check status (reads KV, no wallet needed)
bash setup.sh status

# List supported tokens
bash setup.sh tokens

# Verify integrity
bash setup.sh verify
```

**Background run** (setup.sh env doesn't propagate with nohup):
```bash
cd ~/.openclaw/workspace/paper-kv
export $(grep -v '^#' .env | xargs)
nohup python3 -u paper_kv.py > /tmp/paper-kv.log 2>&1 &
# Log: tail -f /tmp/paper-kv.log
```

## Architecture

Single file: `paper_kv.py` — Python 3.8+, stdlib only.

- **Price feed**: NEAR Intents 1Click API (`1click.chaindefuser.com/v0/tokens`) — 156+ tokens, no API key
- **History seeding**: Binance klines for initial momentum data on first tick
- **Storage**: NEAR FastData KV via OutLayer custody wallet (gasless writes)
- **Fallback**: near-cli-rs for KV writes if no OutLayer key

### Key Classes

| Class | Purpose |
|-------|---------|
| `PriceFeed` | Fetches prices from NEAR Intents, seeds from Binance, computes momentum |
| `IntegrityChecker` | Verifies data consistency (positions, trades, balance) |
| `PaperBot` | Main bot — loads state, runs strategy, manages positions, saves to KV |

## Config (.env)

Generated by `setup.sh` on first run. OutLayer API key auto-registered.

| Variable | Default | Description |
|----------|---------|-------------|
| `OUTLAYER_API_KEY` | auto | OutLayer custody wallet key (gasless KV writes) |
| `KV_CONTRACT` | `contextual.near` | FastData KV target |
| `INITIAL_BALANCE` | `10000` | Starting paper balance (USD) |
| `TRADE_SIZE` | `100` | USD per trade |
| `DEFAULT_LEVERAGE` | `5` | Leverage multiplier |
| `MAX_OPEN_TRADES` | `5` | Max concurrent positions |
| `CHECK_INTERVAL_MS` | `60000` | Tick interval |
| `STRATEGY` | `momentum` | Strategy type |
| `MOMENTUM_LOOKBACK_MINUTES` | `30` | Momentum lookback window |
| `MOMENTUM_THRESHOLD_PCT` | `0.5` | % change to trigger trade |
| `TRADE_PAIRS` | `BTC,ETH,SOL,wNEAR` | Comma-separated NEAR Intents symbols |

## KV Storage Layout

Writes to `contextual.near` via `__fastdata_kv` function call.

| Key | Content |
|-----|---------|
| `state` | `{ balance, totalTrades, wins, losses, totalPnl }` |
| `positions` | Array of open positions |
| `trades` | Array of closed trades with PnL |

## Momentum Strategy

1. Fetch prices from NEAR Intents every tick
2. Seed 30min of Binance history on startup
3. % change >= threshold + UP → LONG
4. % change >= threshold + DOWN → SHORT
5. Reversal >= threshold → close

**Liquidation**: Real perp mechanics with maintenance margin.
- Long liq: `entry * (1 - 1/leverage + 0.005)`
- Short liq: `entry * (1 + 1/leverage - 0.005)`
- Funding: 0.01% per 8h, prorated per tick

## Known Issues

- **Binance-seeded candles use `"close"` key**, live prices use `"price"` — `get_momentum()` handles both
- **OutLayer `/wallet/v1/call` needs NEAR for gas** — wallet must be funded (~0.5 NEAR). Unfunded wallets get HTTP 502
- **`setup.sh` env vars don't propagate with `nohup`** — use `export $(grep -v '^#' .env | xargs)` before running
